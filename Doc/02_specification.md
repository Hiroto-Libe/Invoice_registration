# 仕様書（擬似環境向け）

## 1. 文書の目的
- `Doc/01_requirements.md` を実装可能な仕様に具体化する。
- 対象範囲は「Googleフォーム提出（添付あり）→Drive整形保存→Zapier経由でNotion登録」。

## 2. 対象範囲

### 2.1 対象
- Googleフォーム入力の受付
- Googleドライブ上のファイル名整形・移動
- Notionデータベースへの登録
- 最低限の処理ログ記録

### 2.2 対象外
- URL提出の完全自動取得
- 会計計上や承認ワークフロー
- 本番運用向けの高度監視・通知

## 3. システム構成（最小）
- 入力: Googleフォーム
- ファイル保管: Googleドライブ
- 自動処理: Zapier
- 記帳先: Notion Database
- ログ: Googleスプレッドシート（任意だが推奨）

## 4. データ仕様

### 4.1 フォーム入力項目
- 請求日（必須, `YYYY-MM-DD`）
- 人の名前（必須）
- 内容（必須）
- 金額（必須, 数値）
- 添付ファイル（必須, PDF/画像, 1ファイル, 最大100MB）
- アクセス可能URL（任意）

### 4.2 正規化ルール
- `人の名前` に「さん」が未付与の場合は付与する。
- 文字列の前後空白は削除する。
- 禁止文字 `\/ : * ? " < > |` は `-` に置換する。
- `内容` は最大80文字目安で切り詰める。
- 金額は半角数字に正規化し、Notion登録時は数値型として扱う。

## 5. ファイル命名仕様

### 5.1 Drive正式ファイル名
- 形式: `YYYYMMDD_人の名前さん_内容_金額.拡張子`
- `YYYYMMDD` は請求日から生成する。
- 拡張子は元ファイルを維持する。
- 同名ファイルが存在する場合は末尾に受付IDを付与する。
  - 例: `YYYYMMDD_人の名前さん_内容_金額_A1B2.pdf`

### 5.2 Notionタイトル
- 形式: `人の名前さん_内容`
- 日付・金額はタイトルに含めない。

## 6. 処理フロー仕様

### 6.1 基本ルート（添付あり）
1. ユーザーがGoogleフォームを送信する。
2. 添付ファイルがフォームのアップロード先フォルダに保存される。
3. Zapierで以下を実行する。
   - フォーム回答を取得
   - ファイル情報取得（ファイルID、拡張子、元URL）
   - 正式ファイル名を生成
   - ファイルをリネーム
   - 最終保管フォルダへ移動
4. 最終保管フォルダへの作成/更新をトリガーにNotionへページ作成する。
5. ログに処理結果（成功/失敗）を記録する。

### 6.2 例外ルート（URL提出）
- 擬似環境では自動取得しない。
- URLが入力されている場合、Notionの補助プロパティに格納し、`ステータス=要対応` とする。

## 7. Zapier実装仕様（最小）

### 7.1 Zap A: 受信・整形・移動
- Trigger: Google Forms（新規回答）
- Action:
  - Formatter（文字列整形・日付整形）
  - Google Drive（ファイル情報取得）
  - Google Drive（ファイル名更新）
  - Google Drive（最終保管フォルダへ移動）
- 出力:
  - 正式ファイル名
  - DriveファイルURL
  - 受付ID（重複回避キー）

### 7.2 Zap B: Notion登録
- Trigger: Google Drive（最終保管フォルダの新規ファイル）
- Action: Notion（Create Database Item）
- マッピング:
  - `Title` ← `人の名前さん_内容`
  - `請求日` ← フォーム請求日
  - `金額` ← 数値化した金額
  - `ファイルURL` ← DriveファイルURL
  - `ステータス` ← `受付済み`（URL入力時は `要対応`）

## 8. 二重処理防止
- 一意キー: `フォーム回答ID` または `受付ID` を使用する。
- Notion登録前に同一キーの既存レコード有無を確認する。
- 既存がある場合は登録をスキップし、ログに `duplicate` を記録する。

## 9. エラー処理
- リネーム失敗: エラーログ記録し処理停止。
- Move失敗: エラーログ記録し処理停止。
- Notion登録失敗: 再試行（Zapier標準リトライ）し、失敗時はログ記録。
- 任意でエラー隔離フォルダへ移動する。

## 10. ログ仕様（スプレッドシート）
- 記録項目:
  - 処理日時
  - 受付ID
  - フォーム回答ID
  - 生成ファイル名
  - Drive URL
  - NotionページID（成功時）
  - ステータス（success / error / duplicate）
  - エラー詳細

## 11. 設定パラメータ
- Google Drive:
  - フォームアップロード先フォルダID
  - 最終保管フォルダID
  - エラー隔離フォルダID（任意）
- Notion:
  - Database ID
  - 各プロパティ名
- Zapier:
  - Trigger接続アカウント
  - Action接続アカウント
  - 各マッピング定義

## 12. 受入試験仕様（最小）
- テストデータ3件（PDF/画像混在）で以下を確認する。
  - Driveに正式名で保存される。
  - 同名時に一意化される。
  - Notionタイトルが `人の名前さん_内容` で登録される。
  - DriveファイルURLがNotionに登録される。
  - 重複送信時に二重登録されない。
